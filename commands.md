# clone
git clone is used to create a complete local copy of an existing Git repository.

# checkout
 how you move between different versions of your project.

# -b
creates a new branch and immediately switches you to it.

# status
It tells you exactly what is happening in your repository right now without changing any of your files.

# push
how you upload your local changes to a remote server.

# -u
a way to "link" your local branch to a specific remote branch.

# pull
It downloads changes from a remote server (like GitHub) and immediately merges them into your local files.

# add
how you tell the system: "Include these changes in my next save."

# commit
a permanent snapshot of your project at a specific point in time.

# -m
stands for message

# -a
stands for all or automatically  stage

# -am
allows you to stage and commit your changes in one single line.

# branch
allows you to step away from the main version of your project to work on a new feature or fix a bug without breaking the code that already works.

# -a
stands for all

# -d
stands for "delete." It is most commonly used with the git branch command to remove a branch you no longer need.

# merge
 the act of taking two different versions of a project (usually from two different branches) and combining them into one.
